UUID — 128-битный глобально уникальный идентификатор; здесь он однозначно помечает пользователей, счета и транзакции.

Архитектура хорошая, она и была реализована.

SOLID в коде:
- Single Responsibility: TransactionWriter отвечает только за сохранение транзакций; UserRepository изолирует доступ к данным; Server обрабатывает логику.
- Open/Closed: можно добавлять новые типы транзакций не меняя ничего существующего.
- Liskov: Конкретные Transaction заменяют абстрактную Transaction в любом месте использования.
- Interface Segregation: В интерфейсах нет лишних методов.
- Dependency Inversion: Server реализует интерфейс, который использует ui.

Паттерны:
- Singleton: Server.
- Factory: TransactionFactory.
- Visitor: TransactionVisitor.
- Command: Все транзакции сабмитятся в Execute.
- Strategy: Все транзакции выполняются стратегией в зависимости от типа.
- Observer: TransactionObserver, логгер который пишет транзакции в консоль.

Атомарность и асинхронность:
- Блокировки на уровне счёта гарантируют атомарность транзакций; трансфер блокирует оба счёта по порядку.
- ExecutorService отправляет запросы на операции в фоне (submitDeposit/submitWithdraw/submitTransfer/submitFreeze).
- JDBC-записи выполняются внутри критической секции, чтобы балансы и записи транзакций оставались согласованными.
